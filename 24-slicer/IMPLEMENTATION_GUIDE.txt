PART 2: KEY CONCEPTS YOU NEED TO UNDERSTAND
================================================================================

Concept 1: Indices (Positions in Sequences)
--------------------------------------------
Every character in a string or element in an array has a position called an 
INDEX. Indices always start counting from 0!

Think of indices like seat numbers in a theater:
  - Seat 0: First position
  - Seat 1: Second position
  - Seat 2: Third position
  ... and so on

Example with "Hello":
  Index: 0 1 2 3 4
  Char:  H e l l o

Example with [10, 20, 30, 40]:
  Index: 0  1  2  3
  Elem:  10 20 30 40

Concept 2: Negative Indices (Counting from the End)
-----------------------------------------------------
Negative indices count backwards from the end of the string or array!

Think of it like standing at the end of a line and counting backwards:
  - Position -1: Last element
  - Position -2: Second to last
  - Position -3: Third from last
  ... and so on

Example with "Hello":
  Index:  0 1 2 3 4  (forward)
  Index: -5-4-3-2-1  (backward)
  Char:  H e l l o

This means index -1 refers to the same character as index 4 (the last 'o')

Concept 3: The Starting and Ending Parameters
-----------------------------------------------
slice(start, end) creates a "window" to view your data:

START parameter:
- Where your window begins (inclusive ‚Äî this index IS included)
- If negative, count from the end
- If omitted, defaults to 0 (the beginning)
- If greater than the length, returns empty result

END parameter:
- Where your window stops (exclusive ‚Äî this index is NOT included)
- The element AT this index is NOT included in the result
- If negative, count from the end
- If omitted, goes to the end of the sequence
- If greater than the length, uses the length instead

Example: "Hello" (length is 5)
- slice(1, 4) starts at index 1 ('e'), stops BEFORE index 4
  Result: "ell" (includes indices 1, 2, 3 but NOT 4)
- slice(1) starts at index 1, goes to the end
  Result: "ello"

Concept 4: Not Modifying the Original
---------------------------------------
slice() NEVER changes the original string or array. It creates a new copy
with just the portion you asked for.

Think of it like photocopying a page from a book rather than cutting the
actual book. The original stays intact, and you get a fresh copy.

PART 3: STEP-BY-STEP ALGORITHM (Your Thinking Process)
================================================================================

Step 1: Figure Out the Length
     Determine how many characters (string) or elements (array) you're working
     with. You'll need this to handle indices properly.

Step 2: Handle the Start Index
     - If start is undefined, use 0
     - If start is negative, convert it: actual_start = length + start
     - If start is greater than length, use length
     - If start is negative after conversion, use 0

Step 3: Handle the End Index
     - If end is undefined, use the length (this means "go to the end")
     - If end is negative, convert it: actual_end = length + end
     - If end is greater than length, use length
     - If end is less than 0 after conversion, use 0

Step 4: Ensure Valid Boundaries
     - Make sure start doesn't exceed end
     - If start > end, return an empty string or array
     - If start and end are equal, return an empty string or array

Step 5: Extract the Portion
     - Loop from start index to end index (not including end)
     - Collect each character or element
     - Return the result as a new string or array

PART 4: THE COMPLETE SOLUTION WITH COMMENTS
================================================================================

function slice(sequence, start, end) {
  // STEP 1: Get the length of the sequence (string or array)
  const length = sequence.length;
  
  // STEP 2: Handle the start parameter
  // If start is undefined, we start from the beginning (index 0)
  if (start === undefined) {
    start = 0;
  }
  
  // Convert negative start indices to positive
  // Example: in a string of length 5, start=-2 becomes start=3
  if (start < 0) {
    start = Math.max(0, length + start);
  } else {
    // If start is beyond the length, cap it at the length
    start = Math.min(start, length);
  }
  
  // STEP 3: Handle the end parameter
  // If end is undefined, we go all the way to the end
  if (end === undefined) {
    end = length;
  }
  
  // Convert negative end indices to positive
  if (end < 0) {
    end = Math.max(0, length + end);
  } else {
    // If end is beyond the length, cap it at the length
    end = Math.min(end, length);
  }
  
  // STEP 4: Ensure start doesn't exceed end
  // If start is greater than end, there's nothing to extract
  if (start >= end) {
    // Return empty string if it's a string, empty array if it's an array
    return typeof sequence === 'string' ? '' : [];
  }
  
  // STEP 5: Extract the portion and return it
  // For strings, we can use substring, but we're implementing it manually
  // For arrays, we loop and collect elements
  
  if (typeof sequence === 'string') {
    // Build the result character by character
    let result = '';
    for (let i = start; i < end; i++) {
      result += sequence[i];
    }
    return result;
  } else {
    // sequence is an array
    let result = [];
    for (let i = start; i < end; i++) {
      result.push(sequence[i]);
    }
    return result;
  }
}

Example Usage:
--------------

// Example 1: Slicing a string
const text = "JavaScript";
console.log(slice(text, 0, 4));    // "Java"
console.log(slice(text, 4));       // "Script"
console.log(slice(text, -6));      // "Script" (6 chars from the end)
console.log(slice(text, 1, -1));   // "avaScrip" (skip first and last)

// Example 2: Slicing an array
const numbers = [10, 20, 30, 40, 50];
console.log(slice(numbers, 1, 3));    // [20, 30]
console.log(slice(numbers, -2));      // [40, 50]
console.log(slice(numbers, 0, 5));    // [10, 20, 30, 40, 50] (entire array)
console.log(slice(numbers, 2, 2));    // [] (empty, because start === end)

What These Outputs Mean:
------------------------
- slice("JavaScript", 0, 4) returns "Java" because we include indices 0-3
- slice("JavaScript", 4) returns "Script" because we start at 4 and go to end
- slice("JavaScript", -6) counts 6 positions from the end, which is index 4
- slice(numbers, 1, 3) returns [20, 30] by including indices 1-2 only
- slice(numbers, -2) returns the last 2 elements: [40, 50]

PART 5: UNDERSTANDING THE MATH/LOGIC DEEPER
================================================================================

Let's trace through a complex example step-by-step:

EXAMPLE: slice("Hello", -4, 3)
String: "Hello" (length = 5)
Indices: 0:'H', 1:'e', 2:'l', 3:'l', 4:'o'

STEP 1: Get length
  length = 5

STEP 2: Convert negative start
  start = -4
  Since -4 < 0: start = Math.max(0, 5 + (-4)) = Math.max(0, 1) = 1

STEP 3: Handle end parameter
  end = 3 (it's positive, so no conversion needed)
  3 is not greater than length (5), so end stays 3

STEP 4: Compare start and end
  start (1) < end (3)? YES, so we continue

STEP 5: Extract from index 1 to 3 (not including 3)
  Index 1: 'e'
  Index 2: 'l'
  Result: "el"

Another Example: slice([5, 10, 15, 20, 25], 2, -1)
Array: [5, 10, 15, 20, 25] (length = 5)
Indices: 0:5, 1:10, 2:15, 3:20, 4:25

STEP 1: length = 5

STEP 2: Handle start (2)
  2 is positive and less than length, so start = 2

STEP 3: Convert negative end
  end = -1
  Since -1 < 0: end = Math.max(0, 5 + (-1)) = Math.max(0, 4) = 4

STEP 4: Compare
  start (2) < end (4)? YES

STEP 5: Extract from index 2 to 4 (not including 4)
  Index 2: 15
  Index 3: 20
  Result: [15, 20]

Why Each Step Is Necessary:
----------------------------
1. Getting the length is crucial for converting negative indices and validating boundaries
2. Converting negative indices makes them easier to work with in the loop
3. Capping values at length prevents going past the sequence
4. Checking if start >= end prevents infinite loops and handles edge cases
5. The actual extraction uses a simple loop to build the result

Why This Is Better Than Other Approaches:
-------------------------------------------
- It handles BOTH strings and arrays with the same logic
- It properly handles negative indices in both directions
- It handles undefined parameters gracefully with defaults
- It's clear and easy to understand, even for beginners

PART 8: COMMON MISTAKES TO AVOID
================================================================================

‚ùå Mistake 1: Forgetting That End Index Is Exclusive
   Problem: You might think slice(string, 0, 3) includes index 3
   Example: slice("Hello", 0, 3) returns "Hel" NOT "Hell"
   Solution: Remember: end is exclusive (NOT included). The slice stops BEFORE reaching the end index.

‚ùå Mistake 2: Not Converting Negative Indices Correctly
   Problem: If you use -1 directly in a loop, you'll get errors
   Example: Looping to "array[-1]" doesn't work in JavaScript
   Solution: Convert negative indices FIRST using: length + negativeIndex

‚ùå Mistake 3: Modifying the Original Sequence
   Problem: Some students accidentally modify the input instead of returning new data
   Solution: NEVER modify the original. Always create and return a NEW string or array.

‚ùå Mistake 4: Forgetting to Handle the Array vs String Difference
   Problem: Treating arrays and strings identically in the return
   Example: Returning a string when an array was expected
   Solution: Check the type and return the correct type (use typeof or Array.isArray())

‚ùå Mistake 5: Off-by-One Errors
   Problem: Getting indices wrong by one position is super common
   Solution: Always test your boundaries with small examples first. Use console.log to debug!

```javascript
function slice(sequence, start, end) {
  // Determine the length of the sequence (works for both strings and arrays)
  const length = sequence.length;
  
  // Handle the start parameter
  if (start === undefined) {
    start = 0;
  }
  
  if (start < 0) {
    start = Math.max(0, length + start);
  } else {
    start = Math.min(start, length);
  }
  
  // Handle the end parameter
  if (end === undefined) {
    end = length;
  }
  
  if (end < 0) {
    end = Math.max(0, length + end);
  } else {
    end = Math.min(end, length);
  }
  
  // If start is at or beyond end, return empty result
  if (start >= end) {
    return typeof sequence === 'string' ? '' : [];
  }
  
  // Extract and return the sliced portion
  if (typeof sequence === 'string') {
    let result = '';
    for (let i = start; i < end; i++) {
      result += sequence[i];
    }
    return result;
  } else {
    // It's an array
    let result = [];
    for (let i = start; i < end; i++) {
      result.push(sequence[i]);
    }
    return result;
  }
}
```

Troubleshooting:
----------------

Problem: "slice is not defined" error in console
Solution: Make sure you pasted the entire function into the console and pressed
Enter. The function must be defined BEFORE you call it.

Problem: Tests fail for arrays
Solution: Make sure you're returning a new array, not a string. Use Array.push()
to build arrays, not string concatenation.

Problem: Negative indices don't work
Solution: Double-check your conversion formula. For a sequence of length N,
the index at position -x should become N - x.

Problem: Some tests pass but others fail
Solution: This usually means boundary cases (like when start === end) aren't
handled correctly. Review Step 4 of the algorithm.
