What is this assignment about?
--------------------------------
You need to create two special functions that let you:
  1. READ values from an object (this is called "getting")
  2. WRITE or UPDATE values in an object (this is called "setting")

Think of it like a filing cabinet:
  - GET = Opening a drawer and reading a note
  - SET = Opening a drawer, updating the note, and then showing someone what 
          you wrote

Why is this useful?
---------------------
Instead of directly accessing an object like: sourceObject.num = 10
You'll create functions that control HOW we access and modify data. This is 
important for building larger programs where you want to control access to data.

PART 2: KEY CONCEPTS YOU NEED TO KNOW
================================================================================

1. OBJECTS IN JAVASCRIPT
------------------------
An object is a container that holds related data. Each piece of data has a 
"key" (a name) and a "value" (the actual data).

Example:
  const sourceObject = {
    num: 42,           // key: "num",  value: 42
    bool: true,        // key: "bool", value: true
    str: 'some text',  // key: "str",  value: 'some text'
    log: console.log   // key: "log",  value: a function
  }

You can access values two ways:
  sourceObject.num           // Returns: 42
  sourceObject['num']        // Returns: 42 (same thing)


2. FUNCTIONS IN JAVASCRIPT
---------------------------
A function is a reusable block of code that you can call multiple times.

Basic structure:
  function functionName(parameter1, parameter2) {
    // Code goes here
    return result;  // Send back a result
  }

When you call the function:
  functionName(value1, value2);


3. PARAMETERS AND ARGUMENTS
----------------------------
Parameters = what the function expects to receive (like recipe ingredients)
Arguments = what you actually give the function (the real ingredients)

Example:
  function add(a, b) {  // a and b are PARAMETERS
    return a + b;
  }
  
  add(5, 3);            // 5 and 3 are ARGUMENTS


4. THE RETURN STATEMENT
-----------------------
"return" sends a value back from the function to the code that called it.

Example:
  function getDouble(number) {
    return number * 2;
  }
  
  let result = getDouble(5);  // result = 10

PART 4: BUILDING THE GET FUNCTION
================================================================================

What should the GET function do?
---------------------------------
  1. Take ONE parameter: a key (like "num" or "str")
  2. Use that key to look up the value in sourceObject
  3. Return that value

Step-by-step walkthrough:

  Step 1: Create the function and give it a parameter
    function get(key) {
    
  Step 2: Access the object using the key
    Notice: We use sourceObject[key] not sourceObject.key
    Why? Because "key" is a variable, not a fixed property name.
    If we wrote sourceObject.key, it would look for sourceObject.key
    literally, which doesn't exist.
    
  Step 3: Return the value
    return sourceObject[key];


Example in action:
  get('num')        // Returns: 42
  get('bool')       // Returns: true
  get('str')        // Returns: 'some text'


================================================================================
PART 5: BUILDING THE SET FUNCTION
================================================================================

What should the SET function do?
---------------------------------
  1. Take TWO parameters: a key (like "num") and a value (the new data)
  2. Update the sourceObject's property at that key with the new value
  3. Return the value you just set

Step-by-step walkthrough:

  Step 1: Create the function with two parameters
    function set(key, value) {
    
  Step 2: Update the sourceObject
    sourceObject[key] = value;
    This is like opening a drawer and replacing the old note with a new one.
    
  Step 3: Return the value
    return value;
    This confirms what you just set.


Example in action:
  set('num', 100)    // Changes num from 42 to 100, returns: 100
  set('bool', false) // Changes bool from true to false, returns: false
  set('str', 'hi')   // Changes str to 'hi', returns: 'hi'

PART 9: COMMON MISTAKES TO AVOID
================================================================================

âŒ MISTAKE 1: Using dot notation with a variable
  // WRONG:
  sourceObject.key  // This looks for sourceObject.key literally
  
  // CORRECT:
  sourceObject[key] // This uses the KEY variable's value


âŒ MISTAKE 2: Forgetting to return a value
  // WRONG:
  function set(key, value) {
    sourceObject[key] = value;
    // Missing return statement!
  }
  
  // CORRECT:
  function set(key, value) {
    sourceObject[key] = value;
    return value;
  }


âŒ MISTAKE 3: Creating new functions instead of returning
  // WRONG:
  function set(key, value) {
    return function() {  // Creating a function instead of returning value!
      sourceObject[key] = value;
    }
  }
  
  // CORRECT:
  function set(key, value) {
    sourceObject[key] = value;
    return value;      // Return the actual value
  }
