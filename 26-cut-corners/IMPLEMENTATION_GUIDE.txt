Concept 1: Integer and Decimal Parts
-------------------------------------
Every number has two parts:
- INTEGER PART: The whole number (3 in 3.7)
- DECIMAL PART: The fractional portion (0.7 in 3.7)

Think of it like money: $3.70 has $3 (integer) and $0.70 (decimal).

To separate these parts mathematically:
  Integer part = the whole number without the decimal
  Decimal part = what's left after removing the integer part

For 3.7:
  Integer part = 3
  Decimal part = 0.7

For -3.7:
  Integer part = -3
  Decimal part = -0.7 (stays with the sign!)

Concept 2: Positive vs. Negative Rounding
------------------------------------------
Rounding works differently depending on the sign!

POSITIVE NUMBERS (like 3.7):
- UP means larger (3.7 ‚Üí 4 is going UP)
- DOWN means smaller (3.7 ‚Üí 3 is going DOWN)

NEGATIVE NUMBERS (like -3.7):
- UP means less negative, closer to zero (-3.7 ‚Üí -3 is going UP)
- DOWN means more negative, away from zero (-3.7 ‚Üí -4 is going DOWN)

This is crucial for understanding ceil(), floor(), and round()!

Concept 3: The Four Rounding Strategies
----------------------------------------

ROUND (round to nearest):
  Looks at the decimal part:
  - If 0.5 or higher ‚Üí round UP to the next integer
  - If less than 0.5 ‚Üí round DOWN to the lower integer
  
  Examples:
    3.5 ‚Üí 4 (decimal is 0.5, round up)
    3.4 ‚Üí 3 (decimal is 0.4, round down)
    3.6 ‚Üí 4 (decimal is 0.6, round up)
    -3.5 ‚Üí -4 (decimal is -0.5, round away from zero)

CEIL (ceiling = roof):
  Always round UP toward positive infinity:
  
  Examples:
    3.1 ‚Üí 4 (even slightly above, go up)
    3.9 ‚Üí 4 (go up)
    -3.9 ‚Üí -3 (going "up" means less negative)
    -3.1 ‚Üí -3 (going "up")

FLOOR (floor = ground):
  Always round DOWN toward negative infinity:
  
  Examples:
    3.9 ‚Üí 3 (even if close to 4, go down)
    3.1 ‚Üí 3 (go down)
    -3.1 ‚Üí -4 (going "down" means more negative)
    -3.9 ‚Üí -4 (going "down")

TRUNC (truncate = cut off):
  Simply remove the decimal part, like cutting with scissors:
  
  Examples:
    3.9 ‚Üí 3 (cut off the 0.9)
    3.1 ‚Üí 3 (cut off the 0.1)
    -3.9 ‚Üí -3 (cut off the -0.9, keep the sign)
    -3.1 ‚Üí -3 (cut off the -0.1, keep the sign)

Concept 4: Key Math Functions You'll Use
-----------------------------------------
You'll use these JavaScript methods to solve this puzzle:

- Math.abs(n): Returns the absolute value (removes negative sign)
  Examples: Math.abs(3.7) = 3.7, Math.abs(-3.7) = 3.7

- Math.sign(n): Returns 1 if positive, -1 if negative, 0 if zero
  Examples: Math.sign(3.7) = 1, Math.sign(-3.7) = -1, Math.sign(0) = 0

- Addition and multiplication: Basic math operators to rebuild the number

These are your tools for solving the puzzle without forbidden techniques!

PART 3: STEP-BY-STEP ALGORITHM (Your Thinking Process)

GENERAL STRATEGY:
Rather than manipulating strings or bits, we'll use pure math:
1. Work with the absolute value (remove negative sign)
2. Extract the integer and decimal parts mathematically
3. Apply the rounding logic
4. Reapply the sign if the number was negative

STEP-BY-STEP FOR EACH FUNCTION:

FLOOR Implementation:
Step 1: Get the absolute value of the number (remove the negative sign)
Step 2: Extract the integer part (the whole number portion)
Step 3: Check if the original number was negative
Step 4: If original was negative AND had a decimal part, subtract 1
Step 5: Reapply the negative sign if needed
Step 6: Return the result

CEIL Implementation:
Step 1: Get the absolute value of the number
Step 2: Extract the integer part
Step 3: Check if the original number was negative
Step 4: If the original was positive AND had a decimal part, add 1
Step 5: Reapply the negative sign if needed
Step 6: Return the result

ROUND Implementation:
Step 1: Get the absolute value of the number
Step 2: Extract the integer part
Step 3: Extract the decimal part
Step 4: If decimal part ‚â• 0.5, round up; otherwise round down
Step 5: Reapply the negative sign if needed
Step 6: Return the result

TRUNC Implementation:
Step 1: Get the absolute value of the number
Step 2: Extract just the integer part (drop everything after decimal)
Step 3: Reapply the negative sign if needed
Step 4: Return the result

THE KEY INSIGHT:
In JavaScript, when you subtract two numbers where one is bigger than the
other, you can use the result to detect if there was a decimal part:
  - If integerPart === number, there's no decimal (whole number)
  - If integerPart !== number, there IS a decimal part

Also: You can add or subtract 1 to change rounding direction!

```javascript
// Function 1: FLOOR - Always round down toward negative infinity
function floor(num) {
  // Get the absolute value so we work with the positive version
  const absNum = Math.abs(num);
  
  // Get just the integer part by subtracting the decimal
  const integerPart = absNum - (absNum % 10);
  
  // WAIT! We can't use % operator! Let's use a different approach:
  // We'll create our own way to extract just the whole number part
  
  // Using Math.floor is cheating, so we need another method
  // Let's think: how can we remove the decimal without % or Math.floor?
  // Answer: Use addition and subtraction cleverly!
  
  // This function removes the decimal part
  const getIntegerPart = (n) => {
    // Start from 0 and count up until we can't add 1 without exceeding n
    // OR use the property that subtracting a number from itself is 0
    // The trick: use a large number to force JavaScript to round
    // Even better: iterate to find the whole number
    let whole = 0;
    if (n >= 0) {
      // Count up from 0
      while (whole + 1 <= n) {
        whole += 1;
      }
    } else {
      // For negatives, count down
      while (whole - 1 >= n) {
        whole -= 1;
      }
    }
    return whole;
  };
  
  // Get the integer part of the absolute value
  const absInteger = getIntegerPart(absNum);
  
  // Now apply floor logic:
  // - If positive: return the integer part
  // - If negative: return integer part minus 1 (if there's a decimal)
  const hasDecimal = absNum !== absInteger;
  
  let result = absInteger;
  
  if (num < 0 && hasDecimal) {
    result = absInteger + 1; // more negative
  }
  
  // Reapply the sign
  return num < 0 ? -result : result;
}

// Function 2: CEIL - Always round up toward positive infinity
function ceil(num) {
  const absNum = Math.abs(num);
  
  // Same helper function to get integer part
  const getIntegerPart = (n) => {
    let whole = 0;
    if (n >= 0) {
      while (whole + 1 <= n) {
        whole += 1;
      }
    } else {
      while (whole - 1 >= n) {
        whole -= 1;
      }
    }
    return whole;
  };
  
  const absInteger = getIntegerPart(absNum);
  const hasDecimal = absNum !== absInteger;
  
  let result = absInteger;
  
  // Opposite of floor:
  // - If positive AND has decimal: add 1
  // - If negative: return the integer part
  if (num > 0 && hasDecimal) {
    result = absInteger + 1;
  }
  
  return num < 0 ? -result : result;
}

// Function 3: ROUND - Round to nearest integer
function round(num) {
  const absNum = Math.abs(num);
  
  const getIntegerPart = (n) => {
    let whole = 0;
    if (n >= 0) {
      while (whole + 1 <= n) {
        whole += 1;
      }
    } else {
      while (whole - 1 >= n) {
        whole -= 1;
      }
    }
    return whole;
  };
  
  const absInteger = getIntegerPart(absNum);
  
  // Get the decimal part: the difference between the number and its integer
  const decimalPart = absNum - absInteger;
  
  let result = absInteger;
  
  // If decimal part is 0.5 or higher, round away from zero
  if (decimalPart >= 0.5) {
    result = absInteger + 1;
  }
  
  return num < 0 ? -result : result;
}

// Function 4: TRUNC - Truncate (remove decimal part)
function trunc(num) {
  const absNum = Math.abs(num);
  
  const getIntegerPart = (n) => {
    let whole = 0;
    if (n >= 0) {
      while (whole + 1 <= n) {
        whole += 1;
      }
    } else {
      while (whole - 1 >= n) {
        whole -= 1;
      }
    }
    return whole;
  };
  
  const absInteger = getIntegerPart(absNum);
  
  // Simply return the integer part with the original sign
  return num < 0 ? -absInteger : absInteger;
}
```

```javascript
// BETTER SOLUTION - Much simpler!

// Helper function: Get just the integer part without using Math.floor
// We use a clever trick with toFixed and Number conversion
// NO WAIT - we can't use string conversion!

// The BEST approach: Use the fact that JavaScript can subtract carefully
function floor(num) {
  // Get the integer by repeatedly subtracting 1 until we go below num
  // Actually, there's a better way: use addition and comparison
  
  // For any number, the floor is the largest integer ‚â§ num
  // We can find it by using Math.abs and sign manipulation
  
  const abs = Math.abs(num);
  // Now we need to find the largest integer ‚â§ abs
  // Without loops, we can use: the integer part exists naturally in division
  
  // KEY INSIGHT: Divide by 1, multiply by 1 = no effect mathematically
  // But we need something that extracts integers without %...
  
  // Use the trick: subtract the decimal part!
  // For 3.7: we want to subtract 0.7
  // How? Create a loop that counts up to the number!
  
  let integer = 0;
  while (integer + 1 <= abs) {
    integer += 1;
  }
  
  let result = integer;
  if (num < 0 && num !== result) {
    result += 1;
  }
  
  return num < 0 ? -result : result;
}

function ceil(num) {
  const abs = Math.abs(num);
  
  let integer = 0;
  while (integer + 1 <= abs) {
    integer += 1;
  }
  
  let result = integer;
  if (num > 0 && num !== integer) {
    result += 1;
  }
  
  return num < 0 ? -result : result;
}

function round(num) {
  const abs = Math.abs(num);
  
  let integer = 0;
  while (integer + 1 <= abs) {
    integer += 1;
  }
  
  const decimal = abs - integer;
  let result = integer;
  
  if (decimal >= 0.5) {
    result += 1;
  }
  
  return num < 0 ? -result : result;
}

function trunc(num) {
  const abs = Math.abs(num);
  
  let integer = 0;
  while (integer + 1 <= abs) {
    integer += 1;
  }
  
  return num < 0 ? -integer : integer;
}
```

PART 5: UNDERSTANDING THE MATH/LOGIC DEEPER
================================================================================

Let's trace through an example: rounding -3.7

FLOOR(-3.7):
-----------
Step 1: abs = Math.abs(-3.7) = 3.7
Step 2: Count up: integer = 0, 1, 2, 3 (stop because 4 > 3.7)
Step 3: integer = 3
Step 4: Check: is num < 0? YES (-3.7 < 0)
Step 5: Check: is num !== integer? YES (-3.7 ‚â† 3)
Step 6: So add 1 to result: result = 3 + 1 = 4
Step 7: Reapply sign: -result = -4
Output: -4 ‚úì

Why this works: Floor always goes toward negative infinity. For -3.7, that's
more negative than -3, so we go to -4. The algorithm correctly identifies that
there's a decimal part and adjusts accordingly.

ROUND(3.7):
-----------
Step 1: abs = Math.abs(3.7) = 3.7
Step 2: Count up: integer = 0, 1, 2, 3 (stop)
Step 3: integer = 3
Step 4: decimal = 3.7 - 3 = 0.7
Step 5: Is 0.7 >= 0.5? YES
Step 6: result = 3 + 1 = 4
Step 7: Is num < 0? NO, so return positive result
Output: 4 ‚úì

Why this works: The decimal part 0.7 is more than halfway to the next integer,
so we round up.

CEIL(-3.1):
-----------
Step 1: abs = Math.abs(-3.1) = 3.1
Step 2: Count up: integer = 0, 1, 2, 3 (stop)
Step 3: integer = 3
Step 4: Check: is num > 0? NO (-3.1 is not > 0)
Step 5: So result stays at 3
Step 6: Reapply sign: -result = -3
Output: -3 ‚úì

Why this works: Ceil goes toward positive infinity. For -3.1, that's less
negative than -4, so we go to -3. Since the number is already negative, and
ceil means "toward zero" for negatives, we don't add more.

TRUNC(3.9):
-----------
Step 1: abs = Math.abs(3.9) = 3.9
Step 2: Count up: integer = 0, 1, 2, 3 (stop because 4 > 3.9)
Step 3: integer = 3
Step 4: Is num < 0? NO, so return positive result
Output: 3 ‚úì

Why this works: Truncate just cuts off the decimal. The integer part of 3.9
is 3, so we return 3.

KEY INSIGHT - The While Loop:
We use: while (integer + 1 <= abs)
This counts up until we reach the largest whole number that doesn't exceed abs.
- If abs = 3.7: counts to 3 (because 4 > 3.7)
- If abs = 3.0: counts to 3 (because 4 > 3.0)
- If abs = 3.1: counts to 3 (because 4 > 3.1)

This is our way to extract the integer part without using Math.floor(), %, or
string conversion!

PART 8: COMMON MISTAKES TO AVOID
================================================================================

‚ùå Mistake 1: Forgetting that negative floor/ceil work differently
   Problem: Thinking floor(-3.7) = -3 because -3 is closer
   Solution: Remember floor goes toward negative infinity. -4 is more
             negative, so floor(-3.7) = -4!

‚ùå Mistake 2: Not reapplying the sign correctly
   Problem: Getting 3 as answer but forgetting to make it negative
   Solution: Always multiply by -1 or use a ternary: 
             return num < 0 ? -result : result

‚ùå Mistake 3: Mixing up the positive/negative logic for ceil
   Problem: Adding 1 to negative numbers when you shouldn't
   Solution: Check `if (num > 0 && hasDecimal)` not `if (num < 0 ...)`

‚ùå Mistake 4: Using string conversion or other forbidden methods
   Problem: Using parseFloat(), substring(), or bitwise operations
   Solution: Stick to pure math with +, -, *, / and comparisons

‚ùå Mistake 5: Infinite loops in your while loop
   Problem: Loop never exits because condition never becomes false
   Solution: Make sure integer is always increasing and will eventually
             be > abs. Use: while (integer + 1 <= abs)
