PART 2: KEY CONCEPTS YOU NEED TO UNDERSTAND
================================================================================

Concept 1: Objects as References
---------------------------------
In JavaScript, objects are stored in memory, and when you assign an object to 
a variable, you're storing a REFERENCE (like a pointer) to that object, not 
a copy of it.

ANALOGY: Imagine a building (the object) and you have a list of addresses 
(references) that all point to the same building. If someone changes the 
building, everyone who has that address sees the change.

Example:
  let myHouse = { address: "123 Main St" };
  let sameHouse = myHouse;  // sameHouse points to the SAME object
  sameHouse.address = "456 Oak Ave";
  console.log(myHouse.address);  // Output: "456 Oak Ave"
  // Both variables point to the same object!

Concept 2: Object Properties
-----------------------------
Objects in JavaScript are like containers with labeled slots. Each label is 
called a "property" and can hold any value - numbers, strings, other objects, 
or even... the object itself!

ANALOGY: A person is an object. Their properties include: age, name, height, etc.
But what if a person has a property called "myself" that refers back to them?
That's exactly what we're doing here!

Example:
  let person = {
    name: "Alex",
    age: 16,
    favoriteFood: "Pizza"
  };

Concept 3: Self-Referencing (The Key Concept!)
-----------------------------------------------
A self-referencing object is when an object has a property that points back to 
itself. This creates a circular structure.

STEP-BY-STEP BREAKDOWN:
  Step 1: Create an empty object
  Step 2: Create a property on that object
  Step 3: Set that property's value to be the object itself
  Step 4: Now when you access obj.prop, you get back obj
  Step 5: When you access obj.prop.prop, it's still obj!
  Step 6: You could do obj.prop.prop.prop.prop... forever (but don't!)

VISUALIZATION:
  
  circular = {
    circular: [points back to circular object]
  }
  
  So: circular.circular === circular  ‚úì TRUE!

PART 3: STEP-BY-STEP ALGORITHM (Your Thinking Process)
================================================================================

Step 1: Create a new object called "circular"
            Use const circular = {};
            This creates an empty object

Step 2: Add a property named "circular" to this object
            This property needs to be added after the object is created
            (We can't reference the object before it exists!)

Step 3: Set that property's value to point to the object itself
            Use: circular.circular = circular;
            This makes the property reference the object it belongs to

Step 4: Verify that the self-reference works
            Check: circular.circular === circular (should be true)
            Try accessing: circular.circular.circular (should also work!)

Step 5: You're done! The circular object is now complete

PART 4: THE COMPLETE SOLUTION WITH COMMENTS
================================================================================

// Create an empty object and assign it to the variable "circular"
const circular = {};

// Add a property named "circular" to the object that points back to itself
circular.circular = circular;

// That's it! The object now references itself.

Example Usage:
--------------

// Test 1: Check if the property equals the object
console.log(circular.circular === circular);  // Output: true

// Test 2: Try going deeper
console.log(circular.circular.circular === circular);  // Output: true

// Test 3: You could do this 100 times and it's still the same object!
console.log(circular.circular.circular.circular.circular === circular);  
// Output: true

// Test 4: Check the object in the console
console.log(circular);  
// Output: { circular: {...} } where {...} points back to itself

What the Outputs Mean:
----------------------
When you see "true" as the output, it means the references are the same. 
The property "circular" is pointing back to the original object.

When you expand the object in the browser console, you'll see it shows 
"circular: {circular: {circular: {...}}}" because it's trying to display 
the infinite self-reference. The browser is smart and stops at some point 
to avoid showing infinitely nested objects!

PART 5: UNDERSTANDING THE LOGIC DEEPER
================================================================================

Let's trace through what happens:

BEFORE: We create the object
  const circular = {};  
  // At this point: circular = { }  (empty object)

AFTER: We create the self-reference
  circular.circular = circular;
  // At this point: circular = { circular: <reference to circular> }

NOW WHEN WE ACCESS IT:
  ‚Ä¢ circular.circular
    ‚Üí The "circular" property contains a reference to the original circular object
    ‚Üí So this is the same as "circular"

  ‚Ä¢ circular.circular.circular
    ‚Üí First, circular.circular gives us the "circular" object
    ‚Üí Then we access .circular on it again, getting the same object again!
    ‚Üí So this is the same as "circular"

  ‚Ä¢ This pattern continues forever because we're always getting back to the 
    same object!

WHY IS THIS USEFUL?
This pattern is useful when:
1. Creating linked data structures (each node points to the next, and maybe back)
2. Building graphs where nodes reference each other
3. Creating complex relationships between objects
4. Understanding memory and references (this is a learning concept!)

ALTERNATIVE APPROACHES:
You could create this in one line:
  const circular = { circular: null };
  circular.circular = circular;

Or even fancier, some use Object.freeze() to prevent modification:
  const circular = Object.freeze({ circular: this }); // but 'this' context matters

The simplest approach (shown above) is the best for beginners!

PART 8: COMMON MISTAKES TO AVOID
================================================================================

‚ùå Mistake 1: Creating the property before the object exists
   Problem: 
      circular.circular = circular;  // circular doesn't exist yet!
   Solution: 
      const circular = {};           // Create it first
      circular.circular = circular;   // Then reference it

‚ùå Mistake 2: Confusing object reference with object copying
   Problem:
      const circular1 = {};
      const circular2 = circular1;    // This copies the REFERENCE, not the object
      // Both variables point to the same object
   Solution:
      Understand that = with objects creates a reference, not a copy

‚ùå Mistake 3: Thinking you can access the property before assignment
   Problem:
      const circular = { circular: circular };  // circular isn't defined yet!
   Solution:
      Create the object first, then add the property:
      const circular = {};
      circular.circular = circular;

‚ùå Mistake 4: Using var or let instead of const
   Problem:
      let circular = {};
      circular = {};  // Oops, you might reassign it accidentally
   Solution:
      Use const to prevent accidental reassignment

‚ùå Mistake 5: Trying to stringify a circular object without handling it
   Problem:
      JSON.stringify(circular);  // This creates a circular reference error!
   Solution:
      Avoid stringify for circular objects, or use a replacer function
