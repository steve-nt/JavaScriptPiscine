Concept 1: JavaScript Functions
--------------------------------
A function is like a recipe. You give it ingredients (parameters), it does some
work, and returns a result. Here's the basic structure:

    function functionName(parameter1, parameter2) {
      // Do something here
      return result;
    }

Think of it like a kitchen appliance:
  ‚Ä¢ The function name is the appliance (toaster)
  ‚Ä¢ The parameters are the ingredients (bread)
  ‚Ä¢ The return value is the result (toasted bread)

In our case:
    function max(num1, num2) {
      // Compare the two numbers and decide which is bigger
      return theBiggerOne;
    }


Concept 2: The Ternary Operator - Decision Making in One Line
--------------------------------------------------------------
The ternary operator is a shorthand way to make a choice. It's like asking a
yes/no question and having two different answers:

    condition ? valueIfTrue : valueIfFalse

Think of it like a conversation:
  "Is it raining?" 
  YES ‚Üí "Bring an umbrella"
  NO ‚Üí "Bring sunglasses"

In code:
    const whatToBring = isRaining ? "umbrella" : "sunglasses";

For our assignment:
    num1 > num2 ? num1 : num2

This means: "If num1 is greater than num2, return num1, otherwise return num2"

The ternary operator is perfect for comparing two values because:
  ‚Ä¢ It's concise (takes only one line!)
  ‚Ä¢ It's easy to understand once you know the pattern
  ‚Ä¢ It's exactly designed for this kind of decision


Concept 3: Comparison Operators
--------------------------------
To compare numbers, JavaScript gives us these tools:

    >    Greater than         (5 > 3 is true)
    <    Less than           (5 < 3 is false)
    >=   Greater or equal    (5 >= 5 is true)
    <=   Less or equal       (5 <= 3 is false)
    ==   Equal to            (5 == "5" is true)
    ===  Strictly equal      (5 === "5" is false)

For max and min, you only need > (greater than) and < (less than).

Think of these like a seesaw:
    num1 > num2  means "Is num1 higher on the seesaw?"
    num1 < num2  means "Is num1 lower on the seesaw?"


Concept 4: The Return Statement
--------------------------------
The return statement gives your answer back to whoever called the function.
Once you return, the function stops immediately.

    function checkAge(age) {
      if (age >= 18) {
        return "You can vote!";
      }
      return "You're too young to vote";
    }

When you call checkAge(20), it returns "You can vote!" and stops.


Concept 5: How if/else Statements Work (The Long Way)
------------------------------------------------------
Before we use the ternary operator, let's understand the if/else approach:

    function max(num1, num2) {
      if (num1 > num2) {
        return num1;
      } else {
        return num2;
      }
    }

This says: "Check if num1 is bigger. If yes, return num1. If no, return num2."

// FUNCTION 1: Find the Maximum (Largest) of Two Numbers
function max(num1, num2) {
  // Use the ternary operator to compare num1 and num2
  // If num1 is greater than num2, return num1, otherwise return num2
  return num1 > num2 ? num1 : num2;
}

// FUNCTION 2: Find the Minimum (Smallest) of Two Numbers
function min(num1, num2) {
  // Use the ternary operator to compare num1 and num2
  // If num1 is less than num2, return num1, otherwise return num2
  return num1 < num2 ? num1 : num2;
}


Example Usage:
--------------

max(10, 5)      // Returns: 10 (10 is bigger than 5)
max(3, 8)       // Returns: 8 (8 is bigger than 3)
max(-5, -10)    // Returns: -5 (-5 is bigger than -10)
max(7, 7)       // Returns: 7 (they're equal)

min(10, 5)      // Returns: 5 (5 is smaller than 10)
min(3, 8)       // Returns: 3 (3 is smaller than 8)
min(-5, -10)    // Returns: -10 (-10 is smaller than -5)
min(100, 1)     // Returns: 1 (1 is smaller than 100)


Explanation of Results:
-----------------------
When you call max(10, 5):
  1. The function receives num1=10 and num2=5
  2. It checks: Is 10 > 5? YES, it is true
  3. Since the condition is true, it returns 10
  4. The answer 10 is given back to you

When you call min(10, 5):
  1. The function receives num1=10 and num2=5
  2. It checks: Is 10 < 5? NO, it is false
  3. Since the condition is false, it returns 5
  4. The answer 5 is given back to you

PART 8: COMMON MISTAKES TO AVOID
================================================================================

‚ùå Mistake 1: Using Math.max() or Math.min()
   Problem: The assignment specifically says you can't use these built-in
            functions. Plus, you won't learn how they work!
   Solution: Write your own comparison logic using > or < operators.
   Example:
     WRONG:  return Math.max(num1, num2);
     RIGHT:  return num1 > num2 ? num1 : num2;


‚ùå Mistake 2: Comparing the wrong way
   Problem: Writing num1 < num2 in your max function, which returns the
            minimum instead of the maximum
   Solution: Use > for max and < for min. Double-check your operator!
   Example:
     WRONG:  function max(num1, num2) { return num1 < num2 ? num1 : num2; }
     RIGHT:  function max(num1, num2) { return num1 > num2 ? num1 : num2; }


‚ùå Mistake 3: Forgetting the ternary operator syntax
   Problem: Writing something like "if num1 > num2 return num1" which isn't
            valid JavaScript syntax
   Solution: Remember the pattern: condition ? valueIfTrue : valueIfFalse
   Example:
     WRONG:  return if (num1 > num2) num1 else num2;
     RIGHT:  return num1 > num2 ? num1 : num2;


‚ùå Mistake 4: Returning the wrong value
   Problem: After the comparison, returning the opposite value
   Solution: Be careful with your ternary operator - the first value after ?
            is what you return if the condition is TRUE
   Example:
     WRONG:  return num1 > num2 ? num2 : num1;  // This gives minimum!
     RIGHT:  return num1 > num2 ? num1 : num2;  // This gives maximum!


‚ùå Mistake 5: Not testing with negative numbers
   Problem: Your code might work for positive numbers but fail for negatives
   Solution: Always test with negative numbers, decimals, and edge cases
   Example:
     Test these: max(-10, -5), min(-100, -50), max(-1, 1)
     If they work, you're good!


‚ùå Mistake 6: Confusing the function parameters
   Problem: Mixing up which parameter is which
   Solution: When you call max(10, 5), num1=10 and num2=5 (left to right)
   Example:
     max(10, 5) means num1=10, num2=5
     So if 10 > 5, return 10 ‚úì


‚ùå Mistake 7: Case sensitivity in function names
   Problem: Writing "Max" or "MAX" instead of "max"
   Solution: JavaScript is case-sensitive. The function must be lowercase: max
   Example:
     WRONG:  function Max(num1, num2) { ... }  // Function name is wrong
     RIGHT:  function max(num1, num2) { ... }  // Perfect!

Troubleshooting:
----------------

Problem: I get "ReferenceError: max is not defined"
Solution: Make sure you've pasted the function code into the console and
          pressed Enter BEFORE trying to call the function.

Problem: My function returns the wrong answer
Solution: Check your comparison operator. For max, use >. For min, use <.
          Remember the pattern: condition ? ifTrue : ifFalse

Problem: The HTML file won't open
Solution: Make sure you saved it as ".html" (not ".txt").
          Then double-click it or drag it into your browser window.

Problem: My functions work in the console but tests fail
Solution: This usually means the code you're testing is different from what's
          in your actual file. Make sure you copied the correct code.

Problem: I see "Uncaught SyntaxError: Unexpected identifier"
Solution: You probably made a typo. Check:
          - Did you use { } brackets?
          - Did you spell "function" correctly?
          - Did you use ? and : in the ternary operator?
