Concept 1: Multiplication as Repeated Addition
-----------------------------------------------
Multiplication is nothing more than adding a number to itself multiple times!

Think of it like this: 5 √ó 3 means "add 5 together 3 times"
5 √ó 3 = 5 + 5 + 5 = 15

This is the key insight: Instead of using *, we can use a loop to add the 
number repeatedly. For multiply(5, 3), we add 5 three times.

But wait‚Äîwhat if we need to multiply negative numbers? We need to handle that:
‚Ä¢ If both numbers are positive, keep adding
‚Ä¢ If one number is negative, the result is negative
‚Ä¢ If both are negative, the result is positive (negative √ó negative = positive)

Concept 2: Division as Repeated Subtraction
---------------------------------------------
Division is the opposite of multiplication. It's about asking: "How many times 
can I subtract the divisor from the dividend?"

Example: 20 √∑ 4 = 5 because we can subtract 4 from 20 exactly 5 times:
20 - 4 = 16
16 - 4 = 12
12 - 4 = 8
8 - 4 = 4
4 - 4 = 0 (We did this 5 times, so the answer is 5)

Integer division means we only care about the whole number result, not the 
decimal part. The remainder is what we'll use for the modulo function!

Again, we need to handle signs:
‚Ä¢ Positive √∑ Positive = Positive
‚Ä¢ Negative √∑ Positive = Negative
‚Ä¢ Positive √∑ Negative = Negative
‚Ä¢ Negative √∑ Negative = Positive

Concept 3: Modulo (Remainder) - What's Left Over?
---------------------------------------------------
Modulo is the remainder after division. It answers: "After dividing, what 
amount is left over?"

Example: 17 √∑ 5 = 3 with remainder 2, so 17 % 5 = 2

Here's the mathematical relationship:
dividend = divisor √ó quotient + remainder

In our case:
17 = 5 √ó 3 + 2
17 = 15 + 2 ‚úì

This is incredibly useful! Modulo is used to:
‚Ä¢ Check if a number is even (n % 2 === 0)
‚Ä¢ Create repeating patterns (useful in games, animations)
‚Ä¢ Check if numbers are divisible by something

Concept 4: Handling Negative Numbers
-------------------------------------
JavaScript has specific rules for how negative numbers work with these operators:
‚Ä¢ multiply: The sign follows standard multiplication rules
‚Ä¢ divide: The result's sign depends on the operands' signs
‚Ä¢ modulo: The sign of the result follows the sign of the dividend (the first number)

This is important because different programming languages handle negative 
modulo differently!

PART 3: STEP-BY-STEP ALGORITHM (Your Thinking Process)
================================================================================

MULTIPLY Function Algorithm:
----------------------------
Step 1: Determine the sign of the result
        (Is it negative? Positive? This depends on the signs of a and b)

Step 2: Convert both numbers to their absolute values (make them positive)
        (This makes the math easier)

Step 3: Use a loop to add a to itself b times
        (Start with result = 0, then add a, b times)

Step 4: Apply the sign we determined in Step 1
        (If we calculated it should be negative, multiply the result by -1)

Step 5: Return the result


DIVIDE Function Algorithm:
--------------------------
Step 1: Handle the special case where b (divisor) is 0
        (You can't divide by zero! Return an error or Infinity)

Step 2: Determine the sign of the result
        (Is it negative? This depends on the signs of a and b)

Step 3: Convert both numbers to their absolute values
        (Makes repeated subtraction easier)

Step 4: Use a loop to count how many times we can subtract b from a
        (Keep subtracting b from a, counting each subtraction until a < b)

Step 5: Apply the sign we determined in Step 2
        (If the result should be negative, multiply by -1)

Step 6: Return the count (this is our quotient/answer)


MODULO Function Algorithm:
---------------------------
Step 1: Handle the special case where b (divisor) is 0
        (Return NaN or throw an error)

Step 2: Save the sign of a (the dividend)
        (The result's sign should match the dividend's sign)

Step 3: Convert both numbers to their absolute values

Step 4: Use a loop to find the remainder
        (Keep subtracting b from a until a < b. What's left is the remainder)

Step 5: Apply the sign we saved in Step 2
        (If a was negative, make the remainder negative)

Step 6: Return the remainder

// Function 1: Multiplication without using the * operator
function multiply(a, b) {
  // Determine if the result should be negative
  // The result is negative if exactly one of the numbers is negative
  const isNegative = (a < 0) !== (b < 0);
  
  // Convert both numbers to positive to make the algorithm simpler
  a = Math.abs(a);
  b = Math.abs(b);
  
  // Start with zero and add 'a' to itself 'b' times
  let result = 0;
  for (let i = 0; i < b; i++) {
    result = result + a; // Same as: result += a
  }
  
  // If the result should be negative, make it negative
  // Otherwise, return it as is
  return isNegative ? -result : result;
}


// Function 2: Integer division without using the / operator
function divide(a, b) {
  // Special case: Can't divide by zero!
  if (b === 0) {
    return a > 0 ? Infinity : a < 0 ? -Infinity : NaN;
  }
  
  // Determine if the result should be negative
  const isNegative = (a < 0) !== (b < 0);
  
  // Convert both numbers to positive
  a = Math.abs(a);
  b = Math.abs(b);
  
  // Count how many times we can subtract b from a
  let quotient = 0;
  while (a >= b) {
    a = a - b; // Subtract the divisor
    quotient++; // Increase our count
  }
  
  // Apply the sign and return
  return isNegative ? -quotient : quotient;
}


// Function 3: Modulo (remainder) without using the % operator
function modulo(a, b) {
  // Special case: Can't do modulo with zero divisor
  if (b === 0) {
    return NaN;
  }
  
  // Remember the sign of the dividend (first number)
  // The result's sign should match the dividend
  const isNegative = a < 0;
  
  // Convert both numbers to positive for the calculation
  a = Math.abs(a);
  b = Math.abs(b);
  
  // Keep subtracting b from a until a is smaller than b
  // What's left in 'a' is the remainder
  while (a >= b) {
    a = a - b;
  }
  
  // Apply the sign of the original dividend
  return isNegative ? -a : a;
}

In our example:

17 = 17 ‚úì

This is why the three operations are interconnected!

PART 8: COMMON MISTAKES TO AVOID
================================================================================

‚ùå Mistake 1: Forgetting to Handle Negative Numbers
   Problem: Your functions might work for positive numbers but fail for negatives
   Solution: Always check the signs FIRST before doing the math. Remember:
             negative √ó negative = positive
             negative √∑ positive = negative
             The sign of modulo matches the first number's sign

‚ùå Mistake 2: Using the Forbidden Operators
   Problem: Accidentally using *, /, or % in your solution
   Solution: The assignment specifically forbids these! Use only +, -, and loops.
             If you see these operators, delete them and use the loop approach!

‚ùå Mistake 3: Infinite Loops in Division/Modulo
   Problem: If your while loop never stops, the browser freezes
   Reason: Usually because you're not subtracting the divisor correctly
   Solution: Make sure your while condition is "while (a >= b)" not "while (a > b)"
             and always subtract b from a: "a = a - b"

‚ùå Mistake 4: Not Handling Division by Zero
   Problem: Dividing by zero should return Infinity or handle it gracefully
   Solution: Add a check: if (b === 0) return appropriate value
             This prevents errors and crashes

‚ùå Mistake 5: Forgetting to Apply the Sign at the End
   Problem: Your math is right but the sign is wrong (got -15 instead of 15)
   Solution: After doing the math with positive numbers, remember to apply
             the sign you determined at the start:
             return isNegative ? -result : result;

‚ùå Mistake 6: Using Math.imul() or Other Math Methods
   Problem: The assignment says Math.imul is undefined
   Solution: Don't try to use it! Stick to basic arithmetic (+, -)
             and loops. That's the whole point!

  const isNegative = (a < 0) !== (b < 0);

          in divide() and modulo(). Make sure you're actually subtracting
          and that the loop will eventually stop.

Solution: Make sure you're counting iterations, not calculating division.
          The quotient should be a whole number (you count subtractions).
          Check that you initialized quotient to 0 and increment it each loop.
