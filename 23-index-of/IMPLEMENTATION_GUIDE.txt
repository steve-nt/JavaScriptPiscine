PART 1: UNDERSTANDING THE ASSIGNMENT

What You Need to Build:
------------------------
You need to create three functions that search through arrays:

1. **indexOf(array, searchValue, [fromIndex])**
   - Searches from LEFT to RIGHT (start to end)
   - Returns the position (index) where the value is first found
   - If not found, returns -1
   - Can optionally start the search from a specific position

2. **lastIndexOf(array, searchValue, [fromIndex])**
   - Searches from RIGHT to LEFT (end to start)
   - Returns the position (index) where the value is LAST found
   - If not found, returns -1
   - Can optionally start the search from a specific position going backward

3. **includes(array, searchValue, [fromIndex])**
   - Checks if a value EXISTS in the array
   - Returns true if found, false if not found
   - Can optionally start checking from a specific position

IMPORTANT RULE: You CANNOT use the built-in Array methods like indexOf(),
lastIndexOf(), or includes(). You have to write the logic yourself!

Real-World Example:
-------------------
Imagine you have an array of your friends' names:
  friends = ["Alex", "Jordan", "Casey", "Taylor", "Casey", "Morgan"]

indexOf(friends, "Casey")        ‚Üí returns 2 (first "Casey" is at index 2)
lastIndexOf(friends, "Casey")    ‚Üí returns 4 (last "Casey" is at index 4)
includes(friends, "Morgan")      ‚Üí returns true (Morgan is in the list)
includes(friends, "Riley")       ‚Üí returns false (Riley is NOT in the list)

PART 2: KEY CONCEPTS YOU NEED TO UNDERSTAND
================================================================================

Concept 1: Array Indexing
--------------------------
Arrays store items in positions, starting at 0 (not 1!). Think of it like
apartment building numbers:

  Index:     0        1        2        3        4
  Value:   "Alex"  "Jordan" "Casey" "Taylor" "Casey"

To get an item, you use: array[index]
  array[0] = "Alex"
  array[2] = "Casey"

This is crucial because when you find something, you need to return its index!

Concept 2: Loops (Iterating Through Arrays)
--------------------------------------------
A loop lets you check each item one by one. There are different ways:

FOR LOOP (most common for searching):
  for (let i = 0; i < array.length; i++) {
    // i starts at 0, increases by 1 each time, stops before array.length
    // Each time through, i represents the current index
  }

Think of this like walking through the apartment building, checking each unit:
  Unit 0: Check "Alex" - not what I'm looking for
  Unit 1: Check "Jordan" - not what I'm looking for
  Unit 2: Check "Casey" - FOUND IT! Return 2

Concept 3: Comparison (Checking if Two Things Are Equal)
---------------------------------------------------------
To see if something matches what you're searching for, use the equality operator:

  if (array[i] === searchValue) {
    // We found a match!
  }

The === operator checks if two things are EXACTLY the same.
  5 === 5           ‚Üí true
  "hello" === "hello" ‚Üí true
  "hello" === "Hello" ‚Üí false (JavaScript is case-sensitive!)
  5 === "5"         ‚Üí false (number 5 is not the same as string "5")

Concept 4: Understanding "From Index" (Optional Parameter)
-----------------------------------------------------------
Sometimes you don't want to search the ENTIRE array. You want to start from
a specific position. This is why indexOf has that optional third parameter.

  indexOf(array, "Jordan", 2)
  
This means: "Search for Jordan starting from index 2, going forward"

In lastIndexOf, it works backward from that position:
  lastIndexOf(array, "Jordan", 3)
  
This means: "Search for Jordan looking backward from index 3"

Think of it like: "I found Jordan earlier, but I want to see if there's an
earlier (or later for lastIndexOf) version of Jordan in my list."

Concept 5: Return Values Matter
--------------------------------
Your functions MUST return the right thing:
- indexOf and lastIndexOf: return a NUMBER (the index) or -1 if not found
- includes: returns a BOOLEAN (true or false)

If you return the wrong type, your function doesn't work correctly!

PART 3: STEP-BY-STEP ALGORITHM (Your Thinking Process)
================================================================================

FOR indexOf (searching left to right):
--------------------------------------
Step 1: Figure out where to start
        - Use fromIndex if provided, otherwise start at 0
        
Step 2: Create a loop that goes through the array from start to end
        - Loop through each index from our start point
        
Step 3: Check each element as you go
        - Compare array[i] with searchValue using ===
        
Step 4: When you find a match, return that index immediately
        - Don't keep searching, just return!
        
Step 5: If the loop finishes without finding anything, return -1
        - This means the value wasn't in the array

FOR lastIndexOf (searching right to left):
-------------------------------------------
Step 1: Figure out where to start
        - Use fromIndex if provided, otherwise start at array.length - 1
        - (array.length - 1 is the last index)
        
Step 2: Create a loop that goes BACKWARD through the array
        - Loop from our starting point DOWN to 0
        - Decrease i by 1 each time
        
Step 3: Check each element as you go backward
        - Compare array[i] with searchValue using ===
        
Step 4: When you find a match, return that index immediately
        - This is the LAST match because we're searching backward
        
Step 5: If the loop finishes without finding anything, return -1
        - This means the value wasn't in the array

FOR includes (checking if value exists):
-----------------------------------------
Step 1: Figure out where to start
        - Use fromIndex if provided, otherwise start at 0
        
Step 2: Loop through the array from the start point to the end
        
Step 3: Check each element
        - Compare array[i] with searchValue using ===
        
Step 4: If you find a match, return true immediately
        - You found it!
        
Step 5: If the loop finishes, return false
        - It's not in the array

function indexOf(array, searchValue, fromIndex = 0) {
  // Handle case where fromIndex is negative
  // A negative fromIndex means count backward from the end
  if (fromIndex < 0) {
    fromIndex = Math.max(0, array.length + fromIndex);
  }
  
  // Loop from fromIndex to the end of the array
  for (let i = fromIndex; i < array.length; i++) {
    // Check if the current element matches what we're searching for
    if (array[i] === searchValue) {
      // Found it! Return the index right away
      return i;
    }
  }
  
  // If we got here, we never found the value
  return -1;
}

function lastIndexOf(array, searchValue, fromIndex) {
  // If fromIndex is not provided, start from the last element
  if (fromIndex === undefined) {
    fromIndex = array.length - 1;
  }
  
  // Handle negative fromIndex - count backward from the end
  if (fromIndex < 0) {
    fromIndex = array.length + fromIndex;
  }
  
  // Make sure fromIndex doesn't exceed array length
  if (fromIndex >= array.length) {
    fromIndex = array.length - 1;
  }
  
  // Loop BACKWARD from fromIndex to the beginning
  // i-- means decrease i by 1 each time
  for (let i = fromIndex; i >= 0; i--) {
    // Check if the current element matches what we're searching for
    if (array[i] === searchValue) {
      // Found the LAST occurrence! Return it
      return i;
    }
  }
  
  // If we got here, we never found the value
  return -1;
}

function includes(array, searchValue, fromIndex = 0) {
  // Handle negative fromIndex
  if (fromIndex < 0) {
    fromIndex = Math.max(0, array.length + fromIndex);
  }
  
  // Loop from fromIndex to the end
  for (let i = fromIndex; i < array.length; i++) {
    // Check if the current element matches
    if (array[i] === searchValue) {
      // Found it! Return true
      return true;
    }
  }
  
  // Never found it, return false
  return false;
}

--------------
Let's trace through with actual data:

  ‚Üí Starts at index 0, checks: "Alex" (no), "Jordan" (no), "Casey" (YES!)
  ‚Üí Returns 2

lastIndexOf(students, "Casey")
  ‚Üí Starts at index 4, checks: "Casey" (YES!)
  ‚Üí Returns 4

includes(students, "Taylor")
  ‚Üí Checks: "Alex" (no), "Jordan" (no), "Casey" (no), "Taylor" (YES!)
  ‚Üí Returns true

includes(students, "Riley")
  ‚Üí Checks all 5 students, never finds "Riley"
  ‚Üí Returns false

indexOf(students, "Jordan", 2)
  ‚Üí Starts looking from index 2 instead of 0
  ‚Üí Checks: "Casey" (no), "Taylor" (no), "Casey" (no)
  ‚Üí Returns -1 (not found from that point onward)

PART 5: UNDERSTANDING THE MATH/LOGIC DEEPER
================================================================================

Let's trace through indexOf step by step with actual numbers:

Array: [10, 20, 30, 20, 40]
Looking for: 20
Starting from: 0

Loop iteration 1: i = 0
  - Is array[0] === 20?
  - Is 10 === 20? NO
  - Continue to next iteration

Loop iteration 2: i = 1
  - Is array[1] === 20?
  - Is 20 === 20? YES!
  - RETURN 1 immediately (don't keep searching)

Result: 1 (This is the FIRST occurrence)


Let's trace through lastIndexOf with the same array:

Array: [10, 20, 30, 20, 40]
Looking for: 20
Starting from: 4 (the end)

Loop iteration 1: i = 4
  - Is array[4] === 20?
  - Is 40 === 20? NO
  - i-- so i becomes 3

Loop iteration 2: i = 3
  - Is array[3] === 20?
  - Is 20 === 20? YES!
  - RETURN 3 immediately

Result: 3 (This is the LAST occurrence)

Notice the difference! Both functions search for the same value, but:
- indexOf finds the FIRST one (2nd position)
- lastIndexOf finds the LAST one (4th position)

Why Negative fromIndex Works:
-----------------------------
When fromIndex is negative, like -2:

array.length = 5
fromIndex = -2
Actual start position = array.length + fromIndex = 5 + (-2) = 3

So starting at -2 means "start 2 positions before the end" = index 3

This is like saying: "Start searching from the third-to-last element"

PART 8: COMMON MISTAKES TO AVOID
================================================================================

‚ùå Mistake 1: Using the wrong equality operator
   Problem: Using == instead of ===
   Example: if (array[i] == searchValue) works but is unsafe
   Solution: Always use === for exact comparison
   Why: == does type coercion, == might match "5" with 5, === won't

‚ùå Mistake 2: Forgetting to return -1 when not found
   Problem: Your function returns undefined instead of -1
   Example: Just having a loop without a final return statement
   Solution: Make sure you have "return -1;" after your loop
   Why: indexOf and lastIndexOf MUST return -1 when the value isn't found

‚ùå Mistake 3: Returning early prevents finding the LAST occurrence
   Problem: indexOf and lastIndexOf returning the same value
   Example: Both functions returning the first match they find
   Solution: Make lastIndexOf search backward (i-- instead of i++)
   Why: The direction of search determines which occurrence you find first

‚ùå Mistake 4: Not handling the fromIndex parameter correctly
   Problem: Functions that ignore the fromIndex parameter
   Example: Always starting search from 0 or array.length, no matter what
   Solution: Use the fromIndex value as your loop starting point
   Why: The optional parameter lets users start searching from a specific place

‚ùå Mistake 5: Off-by-one errors with backward search
   Problem: lastIndexOf searching from wrong position or searching one too far
   Example: Starting from array.length instead of array.length - 1
   Solution: Remember that array.length - 1 is the last valid index
   Why: If array has 5 items, valid indices are 0,1,2,3,4 (not 5)

‚ùå Mistake 6: Using array.indexOf() or array.includes() in your solution
   Problem: Violating the assignment requirement to not use built-in methods
   Example: return array.indexOf(searchValue);
   Solution: Write the search logic yourself without using built-in methods
   Why: The whole point is to understand HOW these methods work

‚ùå Mistake 7: Not handling edge cases
   Problem: Functions that break with empty arrays or single elements
   Example: Not checking if array.length is 0
   Solution: Test with edge cases and make sure your loops handle them
   Why: Real code needs to handle unusual inputs gracefully

```javascript
function indexOf(array, searchValue, fromIndex = 0) {
  // Handle negative fromIndex
  if (fromIndex < 0) {
    fromIndex = Math.max(0, array.length + fromIndex);
  }
  
  // Search from fromIndex to the end
  for (let i = fromIndex; i < array.length; i++) {
    if (array[i] === searchValue) {
      return i;
    }
  }
  
  return -1;
}

function lastIndexOf(array, searchValue, fromIndex) {
  // Default fromIndex to the last element
  if (fromIndex === undefined) {
    fromIndex = array.length - 1;
  }
  
  // Handle negative fromIndex
  if (fromIndex < 0) {
    fromIndex = array.length + fromIndex;
  }
  
  // Ensure fromIndex is within bounds
  if (fromIndex >= array.length) {
    fromIndex = array.length - 1;
  }
  
  // Search backward from fromIndex to the beginning
  for (let i = fromIndex; i >= 0; i--) {
    if (array[i] === searchValue) {
      return i;
    }
  }
  
  return -1;
}

function includes(array, searchValue, fromIndex = 0) {
  // Handle negative fromIndex
  if (fromIndex < 0) {
    fromIndex = Math.max(0, array.length + fromIndex);
  }
  
  // Search from fromIndex to the end
  for (let i = fromIndex; i < array.length; i++) {
    if (array[i] === searchValue) {
      return true;
    }
  }
  
  return false;
}
```

```javascript
function indexOf(array, searchValue, fromIndex = 0) {
  if (fromIndex < 0) {
    fromIndex = Math.max(0, array.length + fromIndex);
  }
  for (let i = fromIndex; i < array.length; i++) {
    if (array[i] === searchValue) {
      return i;
    }
  }
  return -1;
}

function lastIndexOf(array, searchValue, fromIndex) {
  if (fromIndex === undefined) {
    fromIndex = array.length - 1;
  }
  if (fromIndex < 0) {
    fromIndex = array.length + fromIndex;
  }
  if (fromIndex >= array.length) {
    fromIndex = array.length - 1;
  }
  for (let i = fromIndex; i >= 0; i--) {
    if (array[i] === searchValue) {
      return i;
    }
  }
  return -1;
}

function includes(array, searchValue, fromIndex = 0) {
  if (fromIndex < 0) {
    fromIndex = Math.max(0, array.length + fromIndex);
  }
  for (let i = fromIndex; i < array.length; i++) {
    if (array[i] === searchValue) {
      return true;
    }
  }
  return false;
```
