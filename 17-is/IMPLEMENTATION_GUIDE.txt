PART 2: KEY CONCEPTS YOU NEED TO UNDERSTAND
================================================================================

Concept 1: Primitives vs Objects
---------------------------------
JavaScript has different types of data. Think of them as different categories
of "things" in JavaScript:

PRIMITIVE TYPES (basic building blocks):
  ‚Ä¢ Numbers:    42, 3.14, -5, NaN
  ‚Ä¢ Strings:    "hello", 'world', ""
  ‚Ä¢ Booleans:   true, false
  ‚Ä¢ Undefined:  undefined (a variable with no value yet)
  ‚Ä¢ Null:       null (nothing/empty - but it's its own thing)

OBJECT TYPES (more complex containers):
  ‚Ä¢ Objects:    { name: "John", age: 25 }
  ‚Ä¢ Arrays:     [1, 2, 3, "hello"]
  ‚Ä¢ Functions:  function() { console.log("hi"); }
  ‚Ä¢ Dates, Maps, Sets, etc.

ANALOGY: Think of primitives as individual LEGO bricks (numbers, text, true/false).
Objects are like LEGO structures built FROM multiple bricks - they're more
complex and can contain multiple pieces of information.

Concept 2: The typeof Operator
-------------------------------
The "typeof" operator is your detective tool in JavaScript. When you use it,
it tells you what type a value is:

  typeof 42           // returns "number"
  typeof "hello"      // returns "string"
  typeof true         // returns "boolean"
  typeof undefined    // returns "undefined"
  typeof function(){} // returns "function"
  typeof []           // returns "object" (interesting!)
  typeof {}           // returns "object"

IMPORTANT: typeof has some quirks! Arrays return "object" instead of "array",
and null also returns "object". We'll need tricks to handle these special cases!

Concept 3: Truthy and Falsy Values
----------------------------------
In JavaScript, every value can be treated as either truthy or falsy when used
in a boolean context (like an if statement).

FALSY VALUES (considered as "false"):
  ‚Ä¢ false (the boolean false)
  ‚Ä¢ 0 (the number zero)
  ‚Ä¢ "" (empty string)
  ‚Ä¢ null (nothing)
  ‚Ä¢ undefined (no value)
  ‚Ä¢ NaN (not a number)

Everything else is TRUTHY! (including "0" as a string, "false" as a string)

ANALOGY: Think of truthy values as "things that exist" and falsy values as
"things that don't exist or are empty."

For example:
  if ("hello") { console.log("This runs because strings are truthy!"); }
  if (0) { console.log("This won't run because 0 is falsy"); }
  if (null) { console.log("This won't run because null is falsy"); }

Concept 4: NaN is Special
-------------------------
NaN means "Not-a-Number" and it's one of the weirdest parts of JavaScript!

The strangest thing? NaN !== NaN is actually TRUE! You can't use == or === to
check for NaN. Instead, you MUST use:
  ‚Ä¢ isNaN(value) - checks if something IS NaN
  ‚Ä¢ Number.isNaN(value) - more reliable version

This is a JavaScript quirk you need to remember!

Concept 5: Array Detection Trick
--------------------------------
Since typeof [] returns "object" (misleading!), we need a way to specifically
detect arrays. We can use:
  ‚Ä¢ Array.isArray(value) - the modern, reliable way
  ‚Ä¢ value instanceof Array - another way that works
  ‚Ä¢ Object.prototype.toString.call(value) === '[object Array]' - the hardcore way

PART 4: THE COMPLETE SOLUTION WITH COMMENTS

const is = {};

// is.num: Check if a value is a number (including checking it's not NaN)
is.num = function(value) {
  // typeof returns "number" for numbers, but NaN is also typeof "number"!
  // We use !Number.isNaN(value) to make sure it's a real number
  return typeof value === "number" && !Number.isNaN(value);
};

// is.nan: Check if a value is specifically NaN (Not-a-Number)
is.nan = function(value) {
  // Number.isNaN is the ONLY reliable way to check for NaN
  return Number.isNaN(value);
};

// is.str: Check if a value is a string
is.str = function(value) {
  // Strings have typeof "string" - straightforward!
  return typeof value === "string";
};

// is.bool: Check if a value is a boolean (true or false)
is.bool = function(value) {
  // Booleans have typeof "boolean"
  return typeof value === "boolean";
};

// is.undef: Check if a value is undefined (hasn't been set)
is.undef = function(value) {
  // undefined has typeof "undefined"
  return typeof value === "undefined";
};

// is.def: Check if a value is DEFINED (NOT undefined)
is.def = function(value) {
  // This is the opposite of is.undef
  return typeof value !== "undefined";
};

// is.arr: Check if a value is an array [1, 2, 3]
is.arr = function(value) {
  // We can't use typeof because typeof [] returns "object"
  // Array.isArray() is specifically designed to detect arrays
  return Array.isArray(value);
};

// is.obj: Check if a value is an object or null
is.obj = function(value) {
  // Objects, arrays, and null all have typeof "object"
  // The assignment wants us to return true for all of these!
  return typeof value === "object";
};

// is.fun: Check if a value is a function
is.fun = function(value) {
  // Functions have typeof "function"
  return typeof value === "function";
};

// is.truthy: Check if a value is truthy
is.truthy = function(value) {
  // Convert to boolean by using double negation (!!)
  // First ! converts to boolean and negates
  // Second ! negates again, giving us the boolean value
  return !!value;
};

// is.falsy: Check if a value is falsy
is.falsy = function(value) {
  // Convert to boolean and negate
  // Single ! converts to boolean and negates it
  return !value;
};

Example Usage:
--------------
console.log(is.num(42));              // true - it's a number
console.log(is.num("hello"));         // false - it's a string
console.log(is.nan(NaN));             // true - it's NaN
console.log(is.nan(42));              // false - it's a real number
console.log(is.str("hello"));         // true - it's a string
console.log(is.bool(true));           // true - it's a boolean
console.log(is.undef(undefined));     // true - it's undefined
console.log(is.def(42));              // true - it's defined
console.log(is.arr([1,2,3]));         // true - it's an array
console.log(is.arr({a: 1}));          // false - it's an object, not array
console.log(is.obj({}));              // true - it's an object
console.log(is.obj([1,2]));           // true - arrays are objects too!
console.log(is.obj(null));            // true - null is treated as object
console.log(is.fun(alert));           // true - alert is a function
console.log(is.truthy("hello"));      // true - non-empty strings are truthy
console.log(is.truthy(""));           // false - empty strings are falsy
console.log(is.falsy(0));             // true - 0 is falsy
console.log(is.falsy(1));             // false - 1 is truthy

What the outputs mean:
- true means "yes, this value matches the type I'm checking for"
- false means "no, this value does NOT match the type I'm checking for"

PART 5: UNDERSTANDING THE LOGIC DEEPER
================================================================================

Let's trace through some examples step-by-step:

EXAMPLE 1: is.num(42)
  Step 1: typeof 42 === "number" ‚Üí true
  Step 2: Number.isNaN(42) ‚Üí false
  Step 3: true && !false ‚Üí true && true ‚Üí TRUE ‚úì

EXAMPLE 2: is.num(NaN)
  Step 1: typeof NaN === "number" ‚Üí true (NaN is a number type!)
  Step 2: Number.isNaN(NaN) ‚Üí true
  Step 3: true && !true ‚Üí true && false ‚Üí FALSE ‚úì
  (We exclude NaN from being a "number" because it's special)

EXAMPLE 3: is.arr([1, 2, 3])
  Step 1: Array.isArray([1, 2, 3]) ‚Üí TRUE ‚úì
  (This is specifically designed to detect arrays)

EXAMPLE 4: is.truthy("hello")
  Step 1: !"hello" ‚Üí false (non-empty strings are truthy, so ! makes it false)
  Step 2: !false ‚Üí true ‚Üí TRUE ‚úì
  (The double negation converts it to a proper boolean)

EXAMPLE 5: is.falsy("")
  Step 1: !"" ‚Üí true (empty strings are falsy, so ! makes it true)
  Step 2: true ‚Üí TRUE ‚úì
  (Single negation is all we need for falsy)

WHY EACH STEP IS NECESSARY:
‚Ä¢ We use typeof because it's the built-in JavaScript way to check types
‚Ä¢ We use Number.isNaN for NaN because typeof doesn't distinguish it properly
‚Ä¢ We use Array.isArray for arrays because typeof mistakenly returns "object"
‚Ä¢ We use !! for truthy to ensure we get a proper boolean value
‚Ä¢ These patterns are the JavaScript standards that professionals use

COMPARING APPROACHES:
Instead of Number.isNaN, you COULD use isNaN, but it's less reliable:
  isNaN("5") returns true (tries to convert to number first)
  Number.isNaN("5") returns false (checks specifically for NaN)
  ‚Üí Number.isNaN is better!

Instead of Array.isArray, you COULD use typeof, but it fails:
  typeof [] returns "object" (not helpful!)
  Array.isArray([]) returns true (exactly what we want)
  ‚Üí Array.isArray is the right choice

PART 8: COMMON MISTAKES TO AVOID
================================================================================

‚ùå Mistake 1: Using typeof to check for NaN
   Problem: typeof NaN returns "number", not "nan"!
             So you can't do typeof value === "nan"
   Solution: Always use Number.isNaN(value) to check for NaN

‚ùå Mistake 2: Using typeof to check for arrays
   Problem: typeof [] returns "object", so you can't distinguish arrays
             from regular objects using typeof alone
   Solution: Use Array.isArray(value) - it's specifically designed for this

‚ùå Mistake 3: Confusing null with undefined
   Problem: They're similar but different!
            null is an assignment value (deliberately nothing)
            undefined means a variable hasn't been set yet
            typeof null returns "object" (a famous JavaScript quirk!)
   Solution: Remember: null and undefined are NOT the same
             Use === to check for each specifically

‚ùå Mistake 4: Forgetting to handle NaN in is.num
   Problem: If you only check typeof value === "number"
            then is.num(NaN) would return true!
            But NaN is NOT a real number...
   Solution: Use && !Number.isNaN(value) to exclude NaN

‚ùå Mistake 5: Using !! incorrectly for truthy
   Problem: Some students try !!!(value) or other variations
            This gets confusing and wrong!
   Solution: Use exactly !!value for truthy
             Use exactly !value for falsy

Troubleshooting:
----------------

Problem: I get "Uncaught SyntaxError" when pasting code
Solution: Make sure there are no typos. Copy the code carefully and check
          that all quotes, braces, and parentheses are correct.

Problem: is.num(5) returns false but it should return true
Solution: Check that you have the complete is.num function including the
          && !Number.isNaN(value) part. Both conditions must pass.

Problem: The console isn't showing me output
Solution: Make sure you're looking at the Console tab, not the Elements tab.
          Some browsers call it "Developer Tools" - press F12 or right-click.

Problem: is.arr([]) returns false when it should be true
Solution: Make sure you're using Array.isArray() not typeof.
          typeof [] returns "object" which won't work here.
