Concept 1: Newton's Second Law of Motion (F = ma)
--------------------------------------------------
Newton's Second Law states: "When forces acting on an object are unbalanced, 
the object will accelerate."

Think of pushing a shopping cart:
  â€¢ If you push harder (MORE FORCE) â†’ it accelerates faster
  â€¢ If the cart is heavier (MORE MASS) â†’ it accelerates slower with the same push

The formula is: a = F / m
  â€¢ a = acceleration (what we're calculating)
  â€¢ F = force applied
  â€¢ m = mass of the object

Real-world analogy: It's like dividing the effort (force) by the weight (mass) 
to see how much speedup you get!

Concept 2: Velocity and Time (Speed Changes)
---------------------------------------------
Velocity is how fast something is moving. When velocity changes, the object 
is accelerating!

Think about pressing the gas pedal in a car:
  â€¢ You go from 0 to 60 mph in 6 seconds
  â€¢ The change in speed is 60 mph
  â€¢ The time that passed is 6 seconds
  â€¢ The acceleration is: 60 / 6 = 10 mph per second

The formula is: a = Î”v / Î”t
  â€¢ Î”v (delta v) = final velocity - initial velocity (the CHANGE)
  â€¢ Î”t (delta t) = final time - initial time (the TIME IT TOOK)

Important: Î” means "change in" - it's how much something increased or decreased!

Concept 3: Distance Over Time (Motion Formula)
-----------------------------------------------
When you know how far something traveled and how long it took, you can also 
calculate acceleration!

Think about dropping a ball:
  â€¢ It falls 10 meters in 1 second
  â€¢ We can calculate how fast it's accelerating due to gravity

The formula is: a = 2d / tÂ²
  â€¢ d = distance traveled
  â€¢ t = time elapsed
  â€¢ tÂ² means time multiplied by itself (t Ã— t)

This formula comes from physics, but you just need to know HOW to use it!

Concept 5: Choosing the Right Formula
--------------------------------------
Here's the key insight: You have THREE formulas, but they all need specific data!

Formula 1 (a = F/m):
  â€¢ Needs: force (f) AND mass (m)
  â€¢ Both must exist and be valid

Formula 2 (a = Î”v/Î”t):
  â€¢ Needs: velocity change (Î”v) AND time change (Î”t)
  â€¢ Both must exist and be valid

Formula 3 (a = 2d/tÂ²):
  â€¢ Needs: distance (d) AND time (t)
  â€¢ Both must exist and be valid
  â€¢ t cannot be 0 (you can't divide by 0!)

Your function checks in order: Does this formula work? No? Try the next one!

```javascript
function getAcceleration(obj) {
  // Try Formula 1: a = F / m (Force divided by Mass)
  // This works if we have both force (f) and mass (m)
  if (obj.f !== undefined && obj.m !== undefined) {
    // We have the data we need
    return obj.f / obj.m;
  }
  
  // Try Formula 2: a = Î”v / Î”t (Velocity change divided by Time change)
  // This works if we have both change in velocity and change in time
  if (obj.Î”v !== undefined && obj.Î”t !== undefined) {
    // We have the data we need
    return obj.Î”v / obj.Î”t;
  }
  
  // Try Formula 3: a = 2d / tÂ² (2 times distance divided by time squared)
  // This works if we have both distance (d) and time (t)
  // BUT: time cannot be 0, otherwise we'd divide by zero (impossible!)
  if (obj.d !== undefined && obj.t !== undefined && obj.t !== 0) {
    // Calculate: 2 * distance / (time * time)
    return (2 * obj.d) / (obj.t * obj.t);
  }
  
  // If none of the formulas worked, we don't have enough data
  return "impossible";
}
```

PART 8: COMMON MISTAKES TO AVOID
================================================================================

âŒ Mistake 1: Using 'undefined' Comparison Wrong
   Problem: Writing `if (obj.f)` instead of `if (obj.f !== undefined)`
   Why it's wrong: If f is 0 (a valid value), the condition is false!
   Solution: Always use `if (obj.f !== undefined)` to properly check existence

âŒ Mistake 2: Forgetting to Check for Zero Division
   Problem: Not checking if obj.t !== 0 before using Formula 3
   Why it's wrong: Dividing by 0 causes errors and incorrect results
   Solution: Always include `&& obj.t !== 0` in the Formula 3 condition

âŒ Mistake 3: Checking Formulas in Wrong Order
   Problem: Not stopping after finding a valid formula
   Why it's wrong: Your function might try multiple formulas when one already worked
   Solution: Use `return` immediately when a formula works (this stops the function)

âŒ Mistake 4: Forgetting to Return the String "impossible"
   Problem: Returning null, undefined, or false instead of the string
