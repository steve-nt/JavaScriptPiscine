Concept 1: The Modulo Operator (%)
-----------------------------------
The modulo operator (%) gives you the REMAINDER when one number divides another.

Think of it like this: If you have 17 apples and want to divide them equally
among 5 people, each person gets 3 apples, but there's 1 apple left over.

17 % 5 = 2 (because 17 √∑ 5 = 3 remainder 2)

KEY INSIGHT: If a number is divisible by another number with NO remainder, then
the modulo result is 0!

Examples:
  - 9 % 3 = 0 (9 is divisible by 3)
  - 10 % 5 = 0 (10 is divisible by 5)
  - 15 % 3 = 0 (15 is divisible by 3)
  - 15 % 5 = 0 (15 is divisible by 5)
  - 15 % 15 = 0 (15 is divisible by both!)
  - 7 % 3 = 1 (7 is NOT divisible by 3)
  - 12 % 5 = 2 (12 is NOT divisible by 5)

This is the SECRET WEAPON for solving this problem!

Concept 2: Loops (for loops)
----------------------------
A loop is like giving your computer a repetitive instruction: "Do this task
10 times" instead of writing the same code 10 times.

The syntax for a for loop:

  for (let i = 1; i <= N; i++) {
    // Code here runs N times
  }

Breaking it down:
  - let i = 1: Start with i equal to 1
  - i <= N: Keep looping while i is less than or equal to N
  - i++: After each loop, add 1 to i (++ is short for i = i + 1)

Think of it like counting: "Count from 1 to 10. As you count each number, do
something with it."

Why use loops instead of writing code 100 times? Because:
1. It's WAY shorter
2. You can change one number (N) and the loop automatically handles any size
3. It's less error-prone (fewer chances to make typos)

Concept 3: If/Else If Statements (Conditional Logic)
-----------------------------------------------------
Conditionals let your program make decisions.

The basic structure:

  if (condition1) {
    // Do this if condition1 is TRUE
  } else if (condition2) {
    // Do this if condition2 is TRUE (and condition1 is FALSE)
  } else {
    // Do this if neither condition1 nor condition2 is TRUE
  }

IMPORTANT: For this assignment, the ORDER of your if statements MATTERS!

Why? Because in programming, once a condition is true, the rest are skipped.

For example:
  - Check if divisible by BOTH 3 and 5 first ‚Üí return "NASA"
  - Then check if divisible by 3 ‚Üí return "NA"
  - Then check if divisible by 5 ‚Üí return "SA"
  - Otherwise ‚Üí return the number itself

If you check "divisible by 3" first, when you encounter 15, it will match
"divisible by 3" and return "NA"... but you wanted "NASA"! This is why ORDER
matters.

Concept 4: String Concatenation
--------------------------------
String concatenation means joining strings together.

In JavaScript, you can use the + operator:

  let greeting = "Hello" + " " + "World";
  // Result: "Hello World"

For this assignment, you'll be joining numbers and words separated by spaces:

  let result = "1" + " " + "2" + " " + "NA";
  // Result: "1 2 NA"

There are different ways to do this in JavaScript (like using array.join()),
but for beginners, string concatenation is clearest.

PART 4: THE COMPLETE SOLUTION WITH COMMENTS
================================================================================

Here's the full solution with detailed comments explaining each part:

```javascript
// This is the function definition. It takes one parameter: N
function nasa(N) {
  // Step 1: Create an empty string to store our result
  let result = "";

  // Step 2: Loop from 1 to N (inclusive)
  for (let i = 1; i <= N; i++) {
    // Step 3: Check if the number is divisible by BOTH 3 and 5
    // If a number is divisible by both, it means (number % 3 === 0 AND number % 5 === 0)
    if (i % 3 === 0 && i % 5 === 0) {
      result += "NASA"; // Add "NASA" to the result
    }
    // Step 4: Check if the number is divisible by only 3
    // We use "else if" so this only checks if the previous condition was false
    else if (i % 3 === 0) {
      result += "NA"; // Add "NA" to the result
    }
    // Step 5: Check if the number is divisible by only 5
    else if (i % 5 === 0) {
      result += "SA"; // Add "SA" to the result
    }
    // Step 6: If none of the above, the number isn't divisible by 3 or 5
    else {
      result += i; // Add the number itself
    }

    // Step 7: Add a space after each item (except we'll handle the last one)
    // The ternary operator (? :) is like a mini if/else:
    // "If i is not equal to N, add a space; otherwise add nothing"
    if (i !== N) {
      result += " ";
    }
  }

  // Step 8: Return the final result
  return result;
}
```

  console.log(nasa(3));
  // Output: "1 2 NA"

  console.log(nasa(1));
  // Output: "1"

Each time you call nasa() with a different number, you get a different result!
The function works for any positive integer you give it.

PART 5: UNDERSTANDING THE LOGIC DEEPER
================================================================================

Let's trace through nasa(15) step by step to really understand what happens:

Loop Iteration 1: i = 1
  - Is 1 divisible by both 3 and 5? 1 % 3 = 1, 1 % 5 = 1 ‚Üí NO
  - Is 1 divisible by 3? 1 % 3 = 1 ‚Üí NO
  - Is 1 divisible by 5? 1 % 5 = 1 ‚Üí NO
  - Add "1" to result ‚Üí result = "1"
  - Add space ‚Üí result = "1 "

Loop Iteration 2: i = 2
  - Not divisible by 3 or 5 ‚Üí Add "2" ‚Üí result = "1 2 "

Loop Iteration 3: i = 3
  - Is 3 divisible by both 3 and 5? 3 % 3 = 0, 3 % 5 = 3 ‚Üí NO (must be BOTH)
  - Is 3 divisible by 3? 3 % 3 = 0 ‚Üí YES! Add "NA" ‚Üí result = "1 2 NA "

Loop Iteration 4: i = 4
  - Not divisible by 3 or 5 ‚Üí Add "4" ‚Üí result = "1 2 NA 4 "

Loop Iteration 5: i = 5
  - Is 5 divisible by both 3 and 5? 5 % 3 = 2, 5 % 5 = 0 ‚Üí NO (must be BOTH)
  - Is 5 divisible by 3? 5 % 3 = 2 ‚Üí NO
  - Is 5 divisible by 5? 5 % 5 = 0 ‚Üí YES! Add "SA" ‚Üí result = "1 2 NA 4 SA "

...continuing this pattern...

Loop Iteration 15: i = 15
  - Is 15 divisible by both 3 and 5? 15 % 3 = 0, 15 % 5 = 0 ‚Üí YES!
    Add "NASA" ‚Üí result = "1 2 NA 4 SA NA 7 8 NA SA 11 NA NA 14 NASA"
  - Since i === N (15 === 15), don't add a space

Final result: "1 2 NA 4 SA NA 7 8 NA SA 11 NA NA 14 NASA"

Why the ORDER matters:
The CRITICAL insight is that we check "divisible by both 3 and 5" FIRST.

Why? Because if 15 was checked against "divisible by 3" first, it would return
"NA" and never check the next conditions. But since we check "divisible by both"
first, we catch the case where we need "NASA"!

This is called PRIORITY or PRECEDENCE in programming‚Äîthe order of your
conditions determines which rules apply first.

Alternative approaches you could use:
1. String building with array.join() - more advanced but cleaner
2. Recursion instead of loops - creates the same result differently
3. Ternary operators instead of if/else - more compact but harder to read

For beginners, the if/else approach is clearest!

PART 8: COMMON MISTAKES TO AVOID
================================================================================

‚ùå Mistake 1: Checking "divisible by 3" before "divisible by both 3 and 5"
   Problem: When you hit 15, it matches "divisible by 3" first and returns "NA",
            never checking if it's also divisible by 5. You get wrong output.
   Solution: ALWAYS check the most specific condition first (both 3 and 5),
            then less specific conditions (just 3, just 5)

‚ùå Mistake 2: Forgetting spaces between items
   Problem: Your output is "1 2 NA4SA NA 7..." with missing or extra spaces
   Solution: Add a space after EVERY item except the last one
            Use: if (i !== N) { result += " "; }

‚ùå Mistake 3: Using = instead of ===
   Problem: if (i = 5) is ASSIGNMENT, not comparison! This always runs!
   Solution: Use === for comparison: if (i === 5)
            = means "assign this value"
            === means "is this value equal to?"

‚ùå Mistake 4: Loop boundary errors
   Problem: Your loop only goes 1 to N-1, or goes to N+1
   Solution: Use <= for "up to and including N": for (let i = 1; i <= N; i++)
            Remember: i < N goes up to N-1, but i <= N includes N itself

‚ùå Mistake 5: Returning a number instead of a string
   Problem: Your function returns a number, not a string of words/numbers
   Solution: Remember to build result as a STRING and return result (not a number)
            All your additions should be to a string variable: result += ...
