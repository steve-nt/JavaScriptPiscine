Concept 1: Number Signs & Comparisons
--------------------------------------

Every number in the universe falls into THREE categories:

   Positive numbers: 1, 2, 5.5, 100, 999...
      â€¢ Greater than zero (> 0)
      â€¢ Represent "more," "up," or "increase"

   Negative numbers: -1, -2, -5.5, -100, -999...
      â€¢ Less than zero (< 0)
      â€¢ Represent "less," "down," or "decrease"

   Zero: 0
      â€¢ The boundary point
      â€¢ Neither positive nor negative
      â€¢ Represents "nothing" or "neutral"

Analogy: Think of a number line. You're standing at zero. Numbers to the right 
are positive (happy face ðŸ˜Š), numbers to the left are negative (sad face ðŸ˜ž),

JavaScript lets us compare numbers:
   â€¢ 5 > 0     returns true   (5 is greater than 0, so positive)
   â€¢ -5 < 0    returns true   (-5 is less than 0, so negative)
   â€¢ 0 === 0   returns true   (0 equals 0, so it's zero)

Concept 4: Return Values
------------------------

The "return" keyword sends a value BACK from the function to whoever called it.

   function sign(number) {
      if (number > 0) {
         return 1;  // Send back 1 to the caller
      } else if (number < 0) {
         return -1; // Send back -1 to the caller
      } else {
         return 0;  // Send back 0 to the caller
      }
   }

   let result = sign(5);  // result gets 1
   let result = sign(-3); // result gets -1

Without "return", the function result is undefined (nothing).

   â€¢ Return 0

FOR THE sameSign(num1, num2) FUNCTION:

   â€¢ This tells us if num1 is positive (1), negative (-1), or zero (0)

Step 2: Call sign(num2) and store the result
   â€¢ This tells us if num2 is positive (1), negative (-1), or zero (0)

Step 3: Compare the two results
   â€¢ If they're equal (===), the signs are the same â†’ return true
   â€¢ If they're not equal, the signs are different â†’ return false

// FUNCTION 1: sign()
// Purpose: Determine if a number is positive, negative, or zero
// Takes: one parameter (a number)
// Returns: 1 for positive, -1 for negative, 0 for zero

function sign(number) {
   // Check if the number is greater than zero
   if (number > 0) {
      // It's positive! Return 1
      return 1;
   } 
   // Check if the number is less than zero
   else if (number < 0) {
      // It's negative! Return -1
      return -1;
   } 
   // If neither of the above, it must be zero
   else {
      // It's exactly zero! Return 0
      return 0;
   }
}

// FUNCTION 2: sameSign()
// Purpose: Check if two numbers have the same sign
// Takes: two parameters (two numbers)
// Returns: true if same sign, false if different signs

function sameSign(num1, num2) {
   // Get the sign of the first number using our sign() function
   let sign1 = sign(num1);
   
   // Get the sign of the second number using our sign() function
   let sign2 = sign(num2);
   
   // Compare the signs using strict equality (===)
   // === checks if two values are EXACTLY the same
   if (sign1 === sign2) {
      // The signs match! Both are positive, both negative, or both zero
      return true;
   } else {
      // The signs don't match. One is different from the other
      return false;
   }
}

Example Usage:
--------------

// TESTING THE sign() FUNCTION:
console.log(sign(5));      // Output: 1 (5 is positive)
console.log(sign(-3));     // Output: -1 (-3 is negative)
console.log(sign(0));      // Output: 0 (0 is zero)
console.log(sign(100));    // Output: 1 (100 is positive)
console.log(sign(-0.5));   // Output: -1 (-0.5 is negative)

// TESTING THE sameSign() FUNCTION:
console.log(sameSign(5, 10));      // Output: true (both positive)
console.log(sameSign(-5, -10));    // Output: true (both negative)
console.log(sameSign(5, -10));     // Output: false (different signs)
console.log(sameSign(0, 5));       // Output: false (0 vs positive)
console.log(sameSign(-5, 0));      // Output: false (negative vs 0)
console.log(sameSign(0, 0));       // Output: true (both zero)

What the outputs mean:
   â€¢ sign(5) = 1 means "5 is positive"
   â€¢ sign(-3) = -1 means "-3 is negative"
   â€¢ sign(0) = 0 means "0 is zero"
   â€¢ sameSign(5, 10) = true means "5 and 10 have the same sign (both positive)"
   â€¢ sameSign(5, -10) = false means "5 and -10 have different signs"

PART 8: COMMON MISTAKES TO AVOID
================================================================================

âŒ Mistake 1: Using == instead of ===
   Problem: 
      let x = sign(-5);
      if (x == "-1") { }  // Wrong! -1 is a number, not a string
      
   Solution: Always use === for strict comparison
      if (x === -1) { }   // Right! Checks the actual number

   Why: JavaScript's == does "type coercion" (converts types automatically),
        which can lead to VERY confusing bugs. Always use ===!


âŒ Mistake 2: Forgetting to return a value
   Problem:
      function sign(number) {
         if (number > 0) {
            return 1;
         }
         // Oops! No return for other cases
      }
      
      let result = sign(-5);  // result is undefined!
   
   Solution: Make sure EVERY code path returns something
      function sign(number) {
         if (number > 0) {
            return 1;
         } else if (number < 0) {
            return -1;
         } else {
            return 0;
         }
      }


âŒ Mistake 3: Returning the wrong type
   Problem:
      function sign(number) {
         if (number > 0) {
            return "1";  // String, not a number!
         } else if (number < 0) {
            return "-1"; // String, not a number!
         } else {
            return 0;
         }
      }
      
      sameSign(5, 10);  // May not work correctly
   
   Solution: Return numbers, not strings
      function sign(number) {
         if (number > 0) {
            return 1;    // Number!
         } else if (number < 0) {
            return -1;   // Number!
         } else {
            return 0;    // Number!
         }
      }


âŒ Mistake 4: Using Math.sign() when you're not allowed to
   Problem:
